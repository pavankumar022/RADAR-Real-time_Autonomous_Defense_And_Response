RADAR is an autonomous Security Operations Center (SOC) platform that detects, visualizes, and responds to network cyber threats in real time. It runs entirely on your local machine — no Docker, no cloud, no external dependencies.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🌍 **3D Attack Globe** | Interactive real-time globe with Bezier arc animations for every threat |
| 🔴 **Live Network Capture** | Monitors real inbound connections on SSH, RDP, SMB, HTTP ports |
| 🤖 **AI Incident Playbooks** | MITRE ATT&CK-aligned response playbooks generated by Gemini / Claude |
| 📋 **AI Incident Reports** | Full SOC analyst reports with IoCs, attack timelines, escalation notes |
| 🗺️ **MITRE ATT&CK Matrix** | Live-synced matrix showing technique heatmaps from your real traffic |
| 📊 **Log Archive** | Search, filter, and inspect historical alerts by IP, severity, or technique |
| ⚡ **Replay Mode** | Stress-test your SOC with up to 500+ Events Per Second |
| 📡 **Attack Simulation Tools** | Built-in Nmap & SSH brute-force simulators (pure Python, no binaries needed) |
| 🔄 **File / Stream Ingestion** | Ingest NDJSON log files or POST live events from any external tool |

## 🚀 Quick Start

### Step 1 — Clone the repository

```bash
git clone https://github.com/pavankumar022/RADAR-Real-time_Autonomous_Defense_And_Response.git
cd RADAR-Real-time_Autonomous_Defense_And_Response/radar
```

---

### Step 2 — Run first-time setup (one-time only)

**Windows:**
```cmd
setup.bat
```

**Linux / macOS:**
```bash
bash setup.sh
```

This will:
- ✅ Check Python & Node.js versions
- ✅ Create a Python virtual environment (`.venv/`)
- ✅ Install all Python dependencies
- ✅ Install all frontend npm packages
- ✅ Build the React frontend into `backend/dist/`
- ✅ Create a `.env` config file from the template

---

### Step 3 — (Optional) Add your Gemini API key

Open `radar/.env` and set your key:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

> 💡 **The app works in demo mode without any API key** — synthetic events, 3D globe, log archive, and replay all work. AI Playbooks and AI Reports require a key.  
> Get a free Gemini key at: https://aistudio.google.com/apikey

---

### Step 4 — Start RADAR

**Windows:**
```cmd
start.bat
```

**Linux / macOS:**
```bash
bash start.sh
```

---

### Step 5 — Open your browser

```
http://localhost:54321
```

| URL | Description |
|---|---|
| `http://localhost:54321` | Main Dashboard |
| `http://localhost:54321/api/docs` | FastAPI Interactive Docs |
| `http://localhost:54321/api/redoc` | API Reference |

## 🎯 Simulate Attacks (Attack Demo Tools)

Open a **second terminal window** in the `radar/` folder and run any of the built-in simulation tools. No extra tools or binaries needed — pure Python.

### Nmap Port Scan Simulation
Simulates a real multi-port TCP scan across 13 common attack ports:
```bash
# Windows
python attack_tools/run_nmap_scan.py --target 192.168.1.100

# Linux / macOS
python3 attack_tools/run_nmap_scan.py --target 192.168.1.100
```

### SSH Brute-Force Simulation
Simulates a credential stuffing SSH brute-force attack:
```bash
# Windows
python attack_tools/run_ssh_brute.py --target 192.168.1.100

# Linux / macOS
python3 attack_tools/run_ssh_brute.py --target 192.168.1.100
```

> Replace `192.168.1.100` with any IP on your local network (or your own machine's IP).  
> Alerts will appear instantly on the dashboard with globe arc animations.

## 🌐 Live Network Capture

RADAR automatically monitors inbound connections to your machine on security-sensitive ports:

| Port | Protocol | Detection Type |
|---|---|---|
| 22 | SSH | Authentication probes |
| 80 | HTTP | Web traffic probes |
| 443 | HTTPS | TLS traffic probes |
| 445 | SMB | Exploit scanning |
| 3389 | RDP | Remote desktop attempts |
| 8080 | HTTP-Alt | Web application probes |

> **Windows users:** Live capture on ports below 1024 (22, 80, 443, 445) requires running `start.bat` as **Administrator**. Right-click → "Run as administrator".  
> Ports above 1024 work without admin rights.

> **Linux/macOS users:** You may need `sudo bash start.sh` for privileged port capture. Alternatively, install `tcpdump` (`sudo apt install tcpdump`) for full packet capture.

## 🛠️ Troubleshooting

### Python not found
Make sure Python is in your system PATH. During Windows installation, check **"Add Python to PATH"**.

### Node.js not found
Download from https://nodejs.org/ and restart your terminal after installation.

### Port 54321 already in use
Change `BACKEND_PORT=54321` to another port (e.g., `54322`) in your `.env` file, then restart.

### Live capture not working on Windows
Run `start.bat` as **Administrator** (right-click → Run as administrator) for full port capture capability.

### AI Playbooks show "demo mode"
Add your Gemini API key to `.env`:
```env
GEMINI_API_KEY=your_key_here
```
Get a free key at: https://aistudio.google.com/apikey

### Frontend shows blank page after `start.bat`
Run `build_frontend.bat` to rebuild the frontend, then restart with `start.bat`.
